Kubernetes is a container management technology developed in Google lab to manage containerized applications in different kind of environments such as physical, virtual, and cloud infrastructure.

Features of Kubernetes

Following are some of the important features of Kubernetes.

  • Continues development, integration and deployment
  • Containerized infrastructure
  • Application-centric management
  • Auto-scalable infrastructure
  • Environment consistency across development testing and production
  • Loosely coupled infrastructure, where each component can act as a separate unit
  • Higher density of resource utilization
  • Predictable infrastructure which is going to be created

Kubernetes - Cluster Architecture

  • pod: pod is the smallest execution unit in Kubernetes. abstraction over container. usally one application per pod. each pod gets own ip address.
  • service: Permananet ip address. lifecycle of pod and service not connected. it has 2 functionalitys. Permananet ip and load balancer
  • ingress: API object that provides routing rules to manage external users' access to the services
  • configMap: External config of your application. Don't put credentials in configmap
  • secrets: used to store scecrt data. base64 encoded
  • deployment: blue print for pods. Abstraction of pods. Dbs can't be replicated via deployment
  • volumes: storage in local machine or outside of k8 cluster
  • Replica:
  • statefulset: for stateful apps. access the same volume

3 process in node

  • container run time
  • kubelet: interact with both container and node. it starts pod with a container inside
  • kubeproxy: forwards the request

4 process in master

  • API server: cluster gateway, acts as gate keeper for auth
  • schedular: where to put the pod
  • control manager: detect cluster state change
  • etcd: it is a cluster brain. cluster changes get stored in key value pair. Application data not stroed in etcd

production has multiple masters and nodes. it is diffcult to test our app. so we use minikube

minikube: master and node process run in one node

kubectl: enables pods to run in node.
